# Answer Submission History Verification

## Preconditions

- локальная база синхронизирована с актуальной Prisma schema
- проект запущен локально
- есть доступ в админку и тестовый стажёрский токен

## Smoke Check

1. Пройти попытку с SQL-вопросом и выполнить один запрос.
2. Открыть отчёт по попытке в админке.

Ожидаемый результат:
- страница открывается без ошибок
- в строке SQL-вопроса есть раскрытие `История отправок`

## Positive Case

1. На открытом вопросе ввести один текст и сохранить.
2. Изменить текст и сохранить ещё раз.
3. На SQL-вопросе выполнить один корректный запрос.
4. Затем выполнить второй заведомо неверный запрос.
5. Завершить попытку и открыть её в админке.

Ожидаемый результат:
- у открытого вопроса в `История отправок` видно обе версии текста
- у SQL-вопроса в `История отправок` видно оба запуска
- верхняя строка отчёта не ломается и остаётся компактной
- итог SQL остаётся `верно`, если раньше уже был найден корректный запрос

## Negative Cases

1. Открыть вопрос без повторных отправок.

Ожидаемый результат:
- история либо не показывается, либо содержит только одну отправку без ошибок интерфейса

2. Выполнить неверный SQL первым запуском.

Ожидаемый результат:
- история фиксирует ошибочную отправку
- итоговый результат вопроса остаётся `0 баллов`

## Regression Check

1. Открыть Markdown-экспорт той же попытки.

Ожидаемый результат:
- в отчёте есть `#### Submission History`
- каждая отправка отображается отдельным блоком с номером и payload

2. Проверить обычный quiz-вопрос и manual review формы.

Ожидаемый результат:
- навигация по попытке и ручная проверка в админке продолжают работать как раньше
